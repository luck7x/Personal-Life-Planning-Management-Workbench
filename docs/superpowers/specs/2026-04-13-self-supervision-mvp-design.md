# 监督反思工具 MVP 设计稿

## 1. 目标

本次 MVP 的目标是尽快交付一个可本地运行的前端演示原型，用最小实现跑通以下闭环：

1. 用户添加任务
2. 任务从 `Todo` 进入 `Doing`
3. 系统记录开始时间
4. 时间段同步到周视图日历
5. 用户结束任务后进入 `Done` 或 `垃圾篓`
6. 系统记录结束时间与执行时长
7. 用户能在周视图中回看每天时间如何流动

本版只做前端静态原型，不接后端，不做云端同步，不依赖页面常驻。

## 2. 范围

### 2.1 本次实现

- 两个主视图：`任务视图`、`日历视图`
- 本地持久化：浏览器 `localStorage`
- 任务状态流转：`Todo -> Doing -> Done / 垃圾篓`
- `Todo` 跨日自动结转到 `逾期`
- `Doing` 支持跨天持续
- 周视图日历默认按周展示，周一为起始日
- 任务自动生成日历时间块
- 支持手动补记日历时间段
- 分类系统：预设分类 + 自定义分类
- 放弃任务在日历中显示为灰色删除线

### 2.2 明确不做

- 用户登录
- 云端同步
- Cloudflare 接口接入
- 自动后台定时任务
- 统计报表页
- 搜索与复杂筛选
- 多月视图或年视图
- 浏览器通知或系统通知

## 3. 运行方式与项目约束

- 在当前目录下新建一个独立文件夹用于前端测试，不污染其他文件
- 采用 `HTML + CSS + JavaScript` 三文件模式
- 当前阶段不强行创建 conda 环境
- 仅在后续测试或脚本工具确有需要时再创建 conda 环境

## 4. 页面结构

### 4.1 任务视图

任务视图采用纵向分组布局，分组顺序固定为：

1. `备忘 Todo`
2. `Doing`
3. `Done`
4. `逾期`
5. `垃圾篓`

每组包含：

- 分组标题
- 数量标识
- 必要操作入口
- 卡片列表

该布局同时服务桌面端和移动端，不拆成两套页面，只做响应式适配。

### 4.2 日历视图

日历视图独立成一页，默认展示一周，周一为起始日。

- 横轴：周一到周日
- 纵轴：当天时间流
- 时间刻度：按小时显示
- 事件块长度：按真实分钟长度渲染
- 支持上一周 / 本周 / 下一周切换

## 5. 任务模型

任务对象至少包含以下字段：

- `id`
- `title`
- `status`
- `categoryId`
- `importance`
- `createdAt`
- `dueAt`
- `startedAt`
- `endedAt`
- `durationMs`
- `abandonReason`
- `sortOrder`

其中：

- `createdAt` 创建时自动记录，默认不直接展示
- `durationMs` 在任务完成或放弃时写入
- `abandonReason` 仅垃圾篓任务必填

## 6. 分类模型

### 6.1 预设分类

第一版预设分类固定为：

- `工作`
- `学习`
- `生活`
- `兴趣`
- `娱乐`

### 6.2 自定义分类

支持用户新增自定义分类，至少包含：

- `id`
- `name`
- `color`
- `isPreset`

若某分类后续被删除，已有任务不丢失，而是回退为“已失效分类”占位显示。

## 7. 任务视图行为设计

### 7.1 Todo

Todo 用于放当天备忘、待办和准备处理的事项。

新增 Todo 时支持填写：

- 标题
- 分类
- 可选重要程度
- 可选截止日期 / 时间

排序规则：

- 默认新增任务插入顶部，即最新在前
- 支持用户手动拖拽排序
- 支持切换为按重要程度排序或按时间排序

点击 Todo 任务时弹出操作：

- `开始执行任务`
- `直接完成任务`
- `取消`

其中“直接完成任务”定义为：

- 不进入 `Doing`
- 直接写入 `endedAt`
- `startedAt` 置空
- `durationMs` 记为 `0`
- 直接进入 `Done`
- 不生成日历时间块

### 7.2 Doing

从 Todo 进入 Doing 时：

- 写入 `startedAt`
- 任务状态改为 `Doing`
- 自动同步到日历，生成进行中的时间块

Doing 卡片展示：

- 标题
- 分类色
- 开始时间
- 当前已进行时长
- 跨天时显示 `跨天进行中`

点击 Doing 任务时弹出操作：

- `完成任务`
- `放弃任务`
- `取消`

### 7.3 Done

完成任务时：

- 写入 `endedAt`
- 用 `endedAt - startedAt` 计算 `durationMs`
- 任务状态改为 `Done`
- 同步更新日历时间块结束时间

Done 列表归属按 **结束日期** 计算，而不是按开始日期。

若任务是从 Todo 直接完成：

- `startedAt` 为空
- `durationMs = 0`
- 仅进入 `Done`
- 不在日历中生成执行时间块

### 7.4 逾期

逾期用于承接未开始、但已跨日的 Todo 任务。

逾期来源包括：

- 昨天及更早创建、仍留在 `Todo` 的任务
- 已设置截止时间且已超时、仍留在 `Todo` 的任务

以下任务不进入逾期：

- `Doing`
- `Done`
- `垃圾篓`

### 7.5 垃圾篓

放弃任务时必须填写放弃原因。

放弃后：

- 写入 `endedAt`
- 写入 `durationMs`
- 写入 `abandonReason`
- 任务状态改为 `垃圾篓`

## 8. 跨天与结转规则

### 8.1 Doing 跨天

- Doing 任务跨天后继续留在 `Doing`
- 不拆成两条任务记录
- 卡片标记为 `跨天进行中`
- 时长计算一律采用真实时间戳差值，天然支持跨天

### 8.2 Todo 结转

不依赖页面在 `23:59` 必须开着，采用“懒触发结转”策略。

触发时机：

- 页面初始化
- 页面重新获得焦点
- 用户新增、编辑、流转任务时

若检测到日期已变化：

- 留在 `Todo` 的旧任务自动进入 `逾期`
- `Doing` 不动
- `Done` 不动
- `垃圾篓` 不动

## 9. 日历模型

日历记录分为两类：

1. 自动生成的任务时间块
2. 手动补记的时间块

日历记录至少包含：

- `id`
- `taskId`
- `title`
- `categoryId`
- `startAt`
- `endAt`
- `source`
- `isAbandoned`
- `notes`

说明：

- 自动生成任务块时 `taskId` 关联具体任务
- 手动补记时 `taskId` 为空
- `source` 标记 `task` 或 `manual`

## 10. 日历视图行为设计

### 10.1 自动同步

任务进入 `Doing` 后，立即在日历中创建进行中的时间块。

任务进入 `Done` 或 `垃圾篓` 后：

- 更新对应时间块的结束时间
- 形成完整时段记录

### 10.2 手动补记

用户可在日历中手动添加时间段，填写：

- 标题
- 分类
- 开始时间
- 结束时间
- 可选备注

### 10.3 时间重叠

时间块允许重叠。

渲染规则采用局部车道分栏：

- 每天总宽度固定
- 只有重叠时间段并排显示
- 不重叠的块仍占满可用宽度

不采用整天统一缩窄的渲染方式。

### 10.4 跨天显示

底层数据仍保持一条连续记录，渲染时在午夜拆段显示。

例如：

- 原始记录：`23:30 - 次日 01:10`
- 显示效果：
  - 前一天列显示 `23:30 - 24:00`
  - 次日列显示 `00:00 - 01:10`

### 10.5 放弃任务显示

放弃任务在日历中显示为：

- 灰色
- 删除线
- 保留原时间段

## 11. 弹窗设计

### 11.1 新建任务弹窗

字段：

- 标题
- 分类
- 可选重要程度
- 可选截止日期 / 时间

### 11.2 Todo 操作弹窗

操作：

- `开始执行`
- `直接完成`
- `取消`

### 11.3 Doing 操作弹窗

展示：

- 开始时间
- 当前已进行时长
- 分类

操作：

- `完成任务`
- `放弃任务`
- `取消`

### 11.4 放弃确认弹窗

要求：

- 放弃原因必填
- 空值不可提交

### 11.5 手动补记弹窗

字段：

- 标题
- 分类
- 开始时间
- 结束时间
- 可选备注

## 12. 数据流

### 12.1 新建 Todo

- 表单提交
- 写入任务数据
- 重渲染 Todo 列表

### 12.2 Todo -> Doing

- 更新任务状态
- 写入 `startedAt`
- 生成或更新日历时间块
- 重渲染任务区和日历区

### 12.3 Todo -> Done（直接完成）

- 更新任务状态为 `Done`
- 写入 `endedAt`
- 保持 `startedAt` 为空
- 写入 `durationMs = 0`
- 不生成日历时间块
- 重渲染任务区

### 12.4 Doing -> Done / 垃圾篓

- 写入 `endedAt`
- 计算 `durationMs`
- 更新任务状态
- 更新日历时间块
- 重渲染任务区和日历区

### 12.5 页面初始化

- 读取本地存储
- 执行一次跨日结转检查
- 完成首屏渲染

## 13. 错误处理与边界

必须明确处理以下情况：

- 手动补记结束时间不能早于开始时间
- 放弃任务时原因不能为空
- 任务没有分类时回退为默认可用分类
- `startedAt` 缺失时禁止计算完成耗时
- 进行中的任务时长仅作实时展示，不提前固化
- 跨天时长一律基于时间戳差值计算
- 本地存储读取失败时回退到默认空数据结构
- 若本地 JSON 异常，提示“本地数据读取异常，已回退默认状态”

## 14. MVP 验收标准

以下测试全部通过即认为本版 MVP 可接受：

1. 新建 Todo 后刷新页面仍存在
2. Todo 拖拽排序后顺序能保留
3. Todo 开始执行后进入 Doing，并在日历中出现进行中块
4. Doing 完成后进入 Done，耗时计算正确
5. Doing 放弃时必须填写原因，并进入垃圾篓
6. Doing 跨天后仍留在 Doing，并标记跨天进行中
7. 跨天耗时计算正确
8. Todo 旧任务在下一次打开页面时自动进入逾期
9. 手动补记允许与其他时间块重叠，并排显示
10. 放弃任务在日历中显示为灰色删除线
11. 周视图从周一开始
12. 桌面端和移动端均可操作

## 15. 实现建议

虽然本次选的是快速 MVP 路线，但 `app.js` 内部仍建议按职责拆分逻辑块：

- 存储逻辑
- 任务状态逻辑
- 日历逻辑
- 弹窗逻辑
- 渲染逻辑

这样可以在保持单文件 JavaScript 的同时，避免后续接入 Cloudflare 时完全重写。
