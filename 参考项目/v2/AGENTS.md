# 项目 Agent 协作规范

## 基本要求

- 默认使用简体中文沟通。
- 运行环境按 Windows PowerShell 处理。
- 核心实现文件是 `明心台.html`，修改前先读取真实代码和当前浏览器表现。
- 参考项目只借交互和布局思路，不直接搬运整段实现。
- 当前项目的扩展指令文件为 `instruction.md`，前端设计相关任务必须先读取该文件。
- 当前项目的路线图文件为 `TODO.md`，每轮任务开始前必须读取；用户新增反馈后必须先更新 `TODO.md`，再按最新优先级执行。

## ToDo 驱动执行

- `TODO.md` 是后续项目规划和验收的主入口。
- 主 agent 必须先确认 `TODO.md` 中当前最高优先级任务，再拆解执行。
- 完成任务后必须把对应条目标记为 `[x]`，进行中标记为 `[~]`，阻塞标记为 `[!]`。
- 用户提出新需求或反馈时，先把需求沉淀到 `TODO.md`，再进入编码。
- 如果实际实现顺序需要调整，必须同步修改 `TODO.md` 的“执行顺序”。

## Agent Team

### 编码 agent

- 负责一切编码实现、文件修改和本地修复。
- 修改前必须先定位相关函数、状态字段和 DOM 结构。
- 同一轮只改必要区域，避免顺手重构无关模块。
- 前端改动必须优先套用 `frontend-design` skill 的设计约束：控件清晰、信息密度稳定、文本不重叠、不做廉价整页换肤。
- 编码 agent 生成或修改代码前，必须主动使用 `mcp-router` 的 `search_context` 查代码上下文。
- 编码 agent 涉及前端布局、交互、可视状态时，必须读取 `instruction.md` 并积极使用 `frontend-design` skill。

### 审查 agent

- 负责独立审查编码结果。
- 审查重点：行为回归、状态一致性、边界条件、未验证风险、无用兼容路径。
- 审查 agent 默认只读，不直接改文件；修复由编码 agent 汇总后执行。
- 审查 agent 必须对照 `TODO.md` 和用户原始反馈检查是否偏题。
- 审查 agent 需要积极使用 `mcp-router` 的 `search_context` 定位受影响代码，而不是只看 diff。

### 审查和测试 agent

- 负责专门复现用户反馈和验收测试。
- 测试重点：真实浏览器表现、同时进行任务、多计时器、时间块堆叠、移动/窄屏文字是否溢出。
- 测试 agent 默认只读，不直接改文件；输出复现步骤、失败点和验收标准。
- 测试 agent 必须优先用真实浏览器访问页面验证前端行为；后端改动必须跑对应测试。
- 测试 agent 需要把测试结果反馈给主 agent，由主 agent 汇总后决定是否进入下一轮修复。

### 主 agent / 编排 agent

- 主 agent 不默认直接包办所有任务；主要负责拆解、调度、合并结论、最终落盘。
- 可并行时，主 agent 应把编码任务交给编码 agent，把审查和测试交给审查和测试 agent。
- 同一文件同一区域存在写冲突风险时，主 agent 必须串行安排。
- 主 agent 负责保证 `TODO.md`、`AGENTS.md`、实际代码和测试结论保持一致。

## 并行调度规则

- 信息收集、审查、测试可以并行。
- 同一文件同一区域的代码修改必须串行，避免写冲突。
- 主 agent 负责编排、合并结论和最终落盘。
- 若用户指出“页面里还是不对”，必须补真实浏览器 QA，不能只靠静态代码判断。

## MCP 与 Skills

- 生成或修改代码前，优先使用 `mcp-router` 的 `search_context` 获取代码上下文。
- 复杂或模糊需求再考虑提示增强工具；普通优化不滥用。
- 前端界面和交互修复默认引入 `frontend-design` skill。
- 出现 bug 或异常表现时，先使用系统化调试思路：复现、定位、最小修复、验证。
- 涉及时效性技术选型、开源项目、部署方案或第三方服务时，必须使用外部搜索或官方文档核验。
- 每个 agent 都必须遵守 Skills 和 MCP 规则；不能只由主 agent 检索上下文。

## 前端设计 Skill 引入

- 已选用：`frontend-design`
- 来源：`C:\Users\lucky\.codex\skills\frontend-design\SKILL.md`
- 适用原因：当前项目是单文件 HTML + Tailwind CDN + Font Awesome + Chart.js + 原生 JS 的管理台，主要问题是布局、控件密度、文字避让和视觉一致性。
- 不选用：`web-artifacts-builder`，它更适合 React / shadcn / artifact 型项目，不适合当前静态 HTML 工作台。
- 执行方式：涉及页面、组件、交互、时间轴、卡片、表单、按钮、移动端适配时，先按 `instruction.md` 的项目化设计约束执行，再改 `明心台.html`。

## 验收口径

- 同时开始多个任务时，每个任务必须有独立计时器。
- 结束任一任务时，只结束该任务对应计时器，并生成对应专注记录和日程时间块。
- 重叠时间块采用堆叠展示，透明度和层级能区分不同项目。
- 上方时间块不能遮挡下方标题或时间文字；至少 3 个同时段块仍可读。
